## Phase 4 — ChatSection Refactor

**Edit:**
| File | Change |
|------|--------|
| `sections/ChatSection.tsx` | Major rewrite (detailed below) |

### 4a — User Bubble
- Replace `bg-blue-900/40` with `userBubbleClass` (blue left border, surface-1 gradient)
- Sender row: `IconUser` + "You" in `text-user-blue`
- Replace emoji-prefixed resolved chips with colored chips using `chipClass()`:
  - Line → `chipClass("blue")` — ic-blue accent, map pin icon
  - Dataset → `chipClass("amber")` — ic-amber accent, database icon
  - Time → `chipClass()` — default, clock icon
  - Aims → `chipClass("coral")` — ic-coral accent, target icon

### 4b — Delta Detection
- Compare current turn's `schema.line`, `datasets_in_scope`, `time`/`no_time_filter` with previous turn
- Apply `opacity-45` class to unchanged chips
- Show italic diff note when line, dataset & time are all unchanged

### 4c — Manager Card
- Replace `bg-zinc-800` with `managerCardClass` (orange left border, glow shadow)
- Sender row: `IconCheckCircle` in `bg-stage-manager-soft` badge + "Manager" in `text-stage-manager`

### 4d — Plan Mode (from `ui.plan`)
- Show Line (from `schema.line`), Time (from `schema.time`/`no_time_filter`), Aims (from `ui.plan.aims`) as labeled rows
- Show Benefits as a styled bullet list with orange dots
- Triggered when `ui.plan?.aims?.length && !ui.proposals?.length`

### 4e — Proposals Mode (from `ui.proposals`)
- Intro text: "Here are N analysis options for {line}. Select one to proceed:"
- Render each proposal as `OptionCard` sub-component:
  - Numbered `stage-manager` chip
  - Title
  - Aims/description text
  - Datasets + Join metadata
  - You-might-see italic quote
  - Hover: orange border highlight, "select →" hint, subtle lift
  - Click: sends `confirm {n}` message
- Triggered when `ui.proposals?.length > 0`

### 4f — Quick-Reply Action Chips
- Plan mode buttons: "Go — proceed" (qrPrimary, sends "go"), "More options", "Change something…"
- Proposals mode buttons: "See more options", "Change something…"
- Fallback: `ui.next_step` rendered as italic text when no structured mode matches

### 4g — Composer End-State
- When `isDone`: render `composerBannerClass` (green bg+tint, checkmark icon + "Session complete — start a new analysis from the top bar") above disabled input + disabled send
- When not done: keep existing form with `btnPrimary` send button

## Edge Cases

- **No previous turn** (turn 0) → no delta detection, all chips at full opacity
- **`ui.plan` with `ui.proposals`** → proposals mode wins (proposals are the actionable decision)
- **No `ui.plan` or `ui.proposals`** → fallback to raw markdown in orange-accented card
- **Option click** sends `confirm {n}` matching the existing `ActionButtons` pattern in OutputSection
- **Proposal with missing fields** → gracefully skips missing datasets/join/you-might-see sections
