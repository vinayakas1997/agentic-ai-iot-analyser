# Issue 6 — Chat Panel Redesign (2026-07-07)

## Overview

The Chat panel has several visual inconsistencies and functional gaps compared to the rest of the redesigned app. The Manager's bubble lacks stage identity, structured responses and proposals appear as raw markdown, quick actions are buried in prose text, resolved context chips repeat at full opacity on every turn, and the composer's end-state is just italic placeholder text.

---

## Identified Issues

### 1. Manager's Bubble Carries No Stage Color

**Location:** `ChatSection.tsx` — Manager bubble

**Current State:**
- Manager messages render in a plain `bg-zinc-800` bubble with a gray "Manager" label
- No orange accent bar, no icon badge, no glow — despite Manager = orange being the established stage color everywhere else

**Issue:**
Every other surface in the app uses stage colors (Manager=orange, Planner=yellow, Execution=green). The chat bubble is the one place where the Manager identity is missing, breaking visual consistency.

---

### 2. Two Different Content Schemas Rendered as Raw Markdown

**Location:** `ChatSection.tsx` — Manager agent text

**Current State:**
- `turn.agent` is rendered as raw markdown regardless of content
- Turn 1 contains plan/prose (Line, Time, Aims, Benefits)
- Turn 2 contains numbered proposals with different fields (Datasets, Join, You might see)
- Same agent, same role, two completely different content shapes

**Issue:**
The user has to re-parse the structure from scratch every turn. Structured data from `turn.ui` (plan, proposals) is available but ignored in the chat rendering.

---

### 3. Proposals Are Plain Text, Not Selectable

**Location:** `ChatSection.tsx` — Proposal rendering

**Current State:**
- Proposals render as markdown paragraph text within the agent's message
- No visual indication they are actionable decisions
- The actual proposal interaction is in the Output section via generic "Confirm 1/2/3" buttons

**Issue:**
Proposals are the key decision point for the user but appear as a wall of text with no tappable affordance.

---

### 4. Quick Actions Buried in Prose ("Reply go / more options / tell me what to change")

**Location:** `ChatSection.tsx` — `next_step` field

**Current State:**
- `turn.ui.next_step` is rendered as a markdown block in a green-bordered box
- Reads as a sentence: "Reply go / more options / tell me what to change"
- Not actionable without reading, processing, and typing

**Issue:**
The user should be able to tap pre-built action chips instead of reading a sentence and typing a response.

---

### 5. Resolved Context Chips Repeat at Full Opacity on Every Turn

**Location:** `ChatSection.tsx` — Resolved chips

**Current State:**
- Each turn shows the full set of resolved context chips (line, datasets, time, aims)
- By turn 2, nothing has changed (same line, same dataset, same time) but chips repeat at full opacity

**Issue:**
Repeating unchanged context adds visual noise. Only the delta is informative.

---

### 6. Composer End-State Is Just Italic Placeholder Text

**Location:** `ChatSection.tsx` — Footer / Composer

**Current State:**
- When `isDone` is true, the input shows italic placeholder "Session complete"
- No banner, no icon, no deliberate empty state — blends in with idle state

**Issue:**
The rest of the app has deliberate end/empty states (cards, banners, pills). The composer's end-state doesn't match.
