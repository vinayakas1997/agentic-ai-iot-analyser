# Plan 3 — UI Fix & Enhancement: Manager Agent Surface (2026-07-07)

## Goals
- Consolidate all Manager decision rendering into a single source-of-truth component
- Add inline-edit affordance for correcting Manager interpretations
- Show traceability from suggested aims to proposals
- Embed per-turn context chips in each chat bubble
- Add sticky pipeline progress cards (Manager/Planner/Execution) in Outputs

## Phase 1 — Types & Foundation

**Edit:**
| File | Change |
|---|---|
| `frontend/src/types/manager.ts` | Expand `planner_payload` type with full handoff fields (`line_name`, `datasets`, `task_definition`, `time_range`, `join_catalog`, `datasets_in_scope`, `datasets_excluded`). Add `selectedProposalIndex` and `proposalProvenance` to `TurnUi`. |

## Phase 2 — Create ManagerDecisionCard (Single Source of Truth)

**Create:**
- `frontend/src/components/ManagerDecisionCard.tsx` — consolidated card showing:
  - **Resolved Line** with match reasoning + ✎ edit button (sends "change line to ...")
  - **Datasets in Scope** as chips + join edges + ✎ edit button (sends "change scope to ...")
  - **Time Filter** with range/"No time filter" + ✎ edit button (sends "change time to ...")
  - **Aims** with provenance badges linking back to suggested aims
  - **Full Handoff Payload** (when `showHandoff=true`) — displays the complete `planner_payload` including datasets, joins, scope, alias

**Key behaviors:**
- `ManagerDecisionCard` is rendered in both ContextSection and OutputSection
- Inline edits send natural-language correction messages that the Manager already understands
- Provenance is computed on the frontend by fuzzy-matching `schema.suggested_aims` against `ui.proposals[].aims`
