### 3c — Add `WaitingOnPlanner` Component
- Renders when `isDone && !plannerStarted`
- Yellow `waitingCardClass` with centered layout
- `IconCheckCircle` in `bg-stage-planner-soft` icon container

- Body: "Manager has scoped the aim. Once Planner builds a query plan, it'll appear here — followed by Execution's results."
- `waitingMetaClass` badge with pulsing dot (CSS animation) + "Planner not yet started"
- `lockedRowClass` with `IconLock` + "Execution results will appear once Planner completes"

### 3d — Redesign `ExecutionProgress` Component
New state machine:

| Condition | State | Render |
|-----------|-------|--------|
| `isDone && !plannerStarted` | Waiting | `WaitingOnPlanner` component |
| No events, not done | Nothing | `return null` |
| `planner.start` / `planner.retry` / `executor.run` | Running | `cardClass` + pulsing dot + status text (yellow/green) |
| `task.complete` + results | Results ready | `resultCardClass` per result — green left border, `resultTagClass` with `resultBadgeClass` (grid icon) + "Execution · {title}", structured JSON preview |
| `task.complete` + no results | Complete | `resultCardClass` with `IconCheckCircle` + "Execution · Complete" |
| `task.failed` | Failed | `cardClass` + red status text |

### 3e — Result Card Framework
Each result from `task.complete` renders inside:
```
┌─ resultCardClass ──────────────────────────┐
│  [resultBadgeClass grid-icon] Execution · {title} │  ← resultTagClass
│  ┌─────────────────────────────────────┐    │
│  │  structured data preview (JSON)     │    │  ← ready for KPI/table/chart
│  └─────────────────────────────────────┘    │
└─────────────────────────────────────────────┘
```
The card structure is set up to receive structured data. When real data arrives, the JSON preview can be replaced with KPIs, tables, and charts without changing the container.

## Edge Cases
- **No turn** → "Send a message to start planning." (unchanged)
- **No ui** → "No snapshot for this step." (unchanged)
- **ui.done + planner.start already fired** → no waiting card, shows planner status instead
- **task.complete with empty results** → shows "Execution · Complete" card without data
- **Multiple result cards** → each result gets its own green-accented card with numbered/unique title
