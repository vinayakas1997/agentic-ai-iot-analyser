# Issue 5 — Enhancement of the Context Side (2026-07-07)

## Overview

The Context panel (sidebar) and ManagerDecisionCard have several visual and functional gaps that reduce clarity when working with multiple datasets and columns. Three specific improvements are needed.

---

## Identified Issues

### 1. Stepper Inactive Steps Lack Visual Identity

**Location:** `ContextSection.tsx` — `Stepper` component

**Current State:**
- Inactive Planner and Execution steps render as `text-stage-planner/50` — essentially invisible text at 50% opacity
- No background or pill shape to indicate the step exists
- Only the active step has visual presence

**Issue:**
Inactive steps should still be visible as stage-colored pills (soft variant) so the user can see the full pipeline even when steps haven't started. The current implementation makes Planner and Execution nearly invisible until activated.

---

### 2. Dataset Chips in ManagerDecisionCard Too Generic

**Location:** `ManagerDecisionCard.tsx` — Datasets section (detail variant)

**Current State:**
- Dataset name chips use `bg-white/[0.05] border border-border-2`
- No visual distinction from other field types (time, line, etc.)
- Blends into the dark surface without meaningful differentiation

**Issue:**
Datasets are a critical piece of context and should be visually distinct. Blue glass morphism (semi-transparent blue tint + backdrop blur) would make them stand out while staying in the dark aesthetic.

---

### 3. Multiple Datasets Have No Selection UI in Columns Table

**Location:** `ContextSection.tsx` — Datasets card + Columns card

**Current State:**
- Datasets card lists all datasets with no interaction
- Columns table shows ALL columns across ALL datasets mixed together
- No way to tell which columns belong to which dataset
- Overflow of 5+ columns shows a static "3 more columns" message with no expand/collapse
- No horizontal scroll for long column names/types

**Issue:**
When the schema has multiple datasets with many columns, the columns table becomes a wall of ungrouped data. Users cannot:
- Select a dataset to see only its columns
- Expand the column list beyond the preview limit
- Scroll horizontally to see all column metadata

---

## Proposed Solutions

### 1. Stepper — Soft Stage-Colored Pills for Inactive Steps

Replace `text-stage-planner/50` with `bg-stage-planner-soft text-stage-planner` (plus corresponding for execution). The inactive step renders as a pill with the stage's soft background color and full-color text, with a small dot indicator. Active steps keep the solid color + glow.

### 2. ManagerDecisionCard — Blue Glass Dataset Chips

Replace `bg-white/[0.05] border-border-2` with `bg-ic-blue-soft/60 backdrop-blur-md border-ic-blue/20` for dataset name chips. Creates a frosted blue glass effect that visually differentiates datasets from other field values.

### 3. ContextSection — Dataset Selection + Filtered Columns + Eye Icon + Horizontal Scroll

- **Dataset selection**: Each dataset name in the Datasets card becomes clickable. Clicking highlights it with an amber-tinted background and filters the Columns table to only that dataset's columns. Clicking again deselects (shows all).
- **Header label**: Columns header shows "Columns — {dataset_name}" when filtered
- **Eye icon toggle**: The static "X more columns" message becomes a clickable `IconEye + "X more columns"` that expands to show all columns, and collapses back on re-click
- **Horizontal scroll**: Table wrapper changed from `overflow-hidden` to `overflow-x-auto` with `min-w-full` on the table so long content scrolls
