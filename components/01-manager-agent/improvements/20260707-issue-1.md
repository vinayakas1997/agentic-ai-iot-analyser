# Issue 1 — UI & Codebase Redundancy Cleanup (2026-07-07)

## Issues Identified

### 1. Two UI paradigms side by side
- **DashboardPage** (`pages/DashboardPage.tsx`) — a legacy simple task submission UI using `api/client.ts` to hit `/task`, `/results`, `/tasks` endpoints
- **WorkspacePage** (`pages/WorkspacePage.tsx`) — the conversational agent workspace using `/manager/sessions/...` endpoints via `api/manager.ts`
- Dashboard was unused by the conversational flow but still imported and rendered in `App.tsx`

### 2. Legacy API client with dead endpoints
- `api/client.ts` contained `submitTask()`, `fetchResults()`, `fetchTasks()` hitting `/task`, `/results`, `/tasks` — these endpoints had no corresponding standalone backend support in the manager agent (leftovers from the parent EDAS project)

### 3. WebSocket disconnected from workspace
- `useWebSocket` hook was only wired on `DashboardPage`
- Workspace had **no real-time updates** — after sending a message to the planner, the user never saw planner progress, query execution, or results
- Backend already broadcasts `planner.start`, `planner.result`, `executor.run`, `task.complete`, `task.failed` via WS — workspace just never listened

### 4. Redundant action buttons in OutputSection
- `"Show suggested aims"` and `"More options"` buttons appeared as catch-alls when no plan/proposals existed — these duplicated the natural chat interaction path and added clutter

### 5. Duplicated constant
- `_SOURCE_LABELS` dict (`"line_name"`, `"synonym"`, `"task_alias"`) was defined identically in both `message_format.py` and `slot_inventory.py`
- The same dict also existed independently in the frontend `ContextSection.tsx`

### 6. Backend legacy routes
- `api/routes/tasks.py` and `api/routes/results.py` were registered in `server.py` but were only relevant to the Dashboard flow — no longer needed in the standalone manager agent

### 7. Unnecessary uiStore complexity
- `uiStore.ts` managed an `AppView` type (`"workspace" | "dashboard"`) with `setView()` — only the workspace view is needed now
